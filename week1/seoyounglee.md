## 문제 1

### 문제 정보

- 문제명: 튜플
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/64065
- 사용한 알고리즘 / 자료구조: set, list

### 처음 접근 방식

문자열을 집합으로 변환하고 각 집합의 원소를 비교하여 튜플의 순서를 구한다. 

### 풀이 방법

1. 문자열의 끝쪽 중괄호를 제거한다.
2. "}{"를 기준으로 문자열을 나누어 집합을 분리한다
3. 각 집합을 원소 개수 기준으로 오름차순한다. 
4. 집합을 순서대로 확인하며 아직 나오지 않은 숫자를 찾는다.
5. 새로운 숫자를 정답 배열에 추가하고 set에 기록한다. 

### 어려웠던 지점

입력값이 중괄호와 쉼표로 구성된 문자열이기 때문에 문자열을 분리할 때 쉼표만 고려하면 집합 내부를 구별하기 어려웠다. 

### 해결 방법

끝 쪽의 괄호들을 제거하고 집합 사이에 무조건 존재하는 "},{"를 기준으로 분리했다. 

groups = s[2:-2].split("},{")

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(L + N log N)
- 공간복잡도: O(L + N)

### 새롭게 알게 된 점

문자열로 주어진 구조는 풀기 편한 형태로 먼저 파싱하는 과정이 필요하다. 그리고 집합 자체에 순서가 없더라고 크기를 이용하면 튜플의 순서 복원이 가능하다. 

이미 등장한 원소를 반복 탐색해야 하는 경우에는 set이 효율적이다. set은 특정 원소가 존재하는지 평균 O(1)에 확인 가능하다.

## 문제 2

### 문제 정보

- 문제명: 문자열 압축
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/60057
- 사용한 알고리즘 / 자료구조: 완전탐색, 구현

### 처음 접근 방식

문자열을 자를 수 있는 모든 가능한 단위 길이를 확인하고자 했다. 문자열 길이가 n일 때 압축 단위는 1부터 n // 2까지만 확인하면 된다. 

### 풀이 방법

1. 원본 문자열의 길이가 초기 최솟값
2. 문자열을 자르는 단위를 1부터 len(s) // 2 까지 변경
3. 첫 문자열 조각은 previous에 저장
4. 다음 문자열 조각을 확인하면서 이전 조각과 같으면 반복 횟수 증가
5. 다른 문자열 조각이 나오면 이전 조각의 길이를 계산
6. 반복 횟수가 1보다 클 때 -> 반복 횟수의 자릿수와 문자열 조각의 길이 합산
7. 반복 횟수가 1 일때 -> 문자열 조각의 길이만 합산
8. 만복문이 끝나면 마지막 조각을 압축 길이에 포함
9. 모든 단위의 압축 길이 중 가장 작은 값을 반환한다. 


### 어려웠던 지점

실제로 문자열을 압축하지 않고 길이만 계산해야 한다는 점

### 해결 방법

변수를 설정해서 압축 결과의 길이만 더했다. 

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(N^2)
- 공간복잡도: O(N)

### 새롭게 알게 된 점

최적의 압축 단위를 한 번에 알 수 없는 경우 가능한 단위를 모두 확인하는 완전 탐색 방식을 활용할 수 있다. 그리고 실제 압축 문자열이 필요한지, 길이만 필요한지 확인하여 효율적인 계산 방식을 찾아야 한다.

## 문제 3

### 문제 정보

- 문제명: 행렬 테두리 회전하기
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/77485
- 사용한 알고리즘 / 자료구조:

### 처음 접근 방식

전체를 회전하는 것이 아니라, 테두리에 있는 숫자만 시계 방향으로 한 칸씩 이동해야 한다. 따라서 위쪽 행을 오른쪽으로 이동/오른쪽 열을 아래로 이동/아래 행을 왼쪽으로 이ㄴ동/왼쪽 열을 위쪽으로 이동하도록 했다.

### 풀이 방법

1. 2차원 행렬을 생성하고, 순서대로 숫자를 채운다
2. 좌표는 1부터 시작하므로, 각 좌표에서 1을 뺀 배열 인덱스로 변환한다.
3. 왼쪽 위부터 임시 변수 previous에 저장한다.
4. 테두리를 시계방향으로 순회하면서 현재 위치 값과 previous를 교환한다.
5. 이동한 값 중 가장 작은 값은 계속 갱신한다.
6. 한 번 회전이 끝나면 최솟값을 정답 배열에 추가한다.
7. 모든 회전을 순서대로 수행한 뒤 정답 배열을 반환한다. 



### 어려웠던 지점

테두리를 순회하는 방향과 반복문의 범위를 정확하게 설정하는 것

### 해결 방법

시계방향과 동일하게 구현한다.

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(RC + Q(R + C))
- 공간복잡도: O(RC + Q)

### 새롭게 알게 된 점

다중할당으로 값을 교환한다.
board[x][y], previous = previous, board[x][y]
